Supabase Implementation Design

Transitioning from LocalStorage to Cloud Persistence

1. Core Architecture

Frontend (index.html)
Supabase JS SDK + Modern UI
⇅ (Real-time Sync)
Database (PostgreSQL)
Tables: shops, products, orders
Storage (Buckets)
Permanent image hosting

2. Data Schema

Tables

  • shops (id, name, image_url, status, owner_email)
  • products (id, shop_id, name, price, image_url)
  • orders (id, customer_details, total, items_json)

Image Handling

  • Replace Base64 strings (slow/large) with Supabase Storage URLs.

3. Key Changes

  • Add Supabase SDK script to <head>
  • Replace saveState() with Supabase .upsert()
  • Replace seedData() with .select() to fetch live data
